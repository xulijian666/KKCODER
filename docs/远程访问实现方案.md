# KKCODER 移动远程访问实现方案

## 1. 项目背景

KKCODER 是一个纯本地 Tauri v2 桌面应用，本方案实现了手机远程操控电脑上的 Claude Code 终端，支持实时对话、会话管理等功能。

### 1.1 核心架构

```
Flutter 手机 App
    ↓ WSS/WS (443/9527)
nginx (SSL 终止，可选)
    ↓ HTTP/WS
axum WebSocket 服务器 (:9527)
    ↓ broadcast 通道
SessionRegistry (DashMap)
    ↓ 订阅输出 / 共享写入
PTY Reader Thread → 前端 Tauri emit + 远程 broadcast
    ↓ Arc<Mutex<Writer>>
Claude Code PTY 进程
```

### 1.2 已实现功能

- 远程终端（实时输入输出）
- 设备配对（PIN + Token，5 分钟过期）
- 基础设置（端口、启用/禁用）
- WebSocket 双向通信（输入、输出、resize）
- 会话列表（REST API）
- 断线重连（seq + ring buffer + replay）
- frp 客户端集成（配置生成、进程管理）
- frp TLS 加密支持
- IP / 域名 / HTTP / HTTPS 全兼容

### 1.3 性能设计

1. **实时链路只走 WebSocket** — 输入输出全在一条连接
2. **每会话独立 broadcast 通道** — 订阅只收本会话消息
3. **16ms 微批处理** — 输出缓冲后批量推送
4. **热路径只走内存** — SQLite 只在握手时读，运行时不查库
5. **seq + ring buffer** — 断线重连可恢复，不丢消息
6. **握手鉴权一次** — 连接内不重复查库

## 2. 实际实现架构

### 2.1 PTY 输出分发

`spawn_terminal` 中 PTY Reader Thread 同时做两件事：
1. 通过 `app_handle.emit("pty-output", ...)` 发送给前端
2. 通过 `broadcast::Sender<OutputFrame>` 发送给远程 WebSocket 客户端

```
PTY Reader Thread
    ├── app_handle.emit("pty-output") → 前端 xterm
    └── output_tx.send(frame) → 远程 WebSocket 客户端
            └── replay.push(frame) → ring buffer（断线重连用）
```

### 2.2 远程输入写入

`ActiveSession.writer` 改为 `Arc<Mutex<Box<dyn Write + Send>>>`，WebSocket Handler 通过 `SessionHandle.pty_writer` 直接写入 PTY，无需经过额外通道。

### 2.3 模块结构

```
src-tauri/src/
  lib.rs                 — 主逻辑 + Tauri 命令 + 服务器启动
  remote/
    mod.rs               — 模块声明
    state.rs             — RemoteServerState, SessionRegistry, SessionHandle,
                           ReplayBuffer, AtomicSessionStatus, DeviceInfo, RemoteConfig
    auth.rs              — AuthToken 提取器, PIN 生成/验证, Token 生成
    handlers.rs          — REST API: 配对, 会话列表, 设备管理, 服务器状态
    ws.rs                — WebSocket 会话处理器（双向通信 + replay）
    server.rs            — axum 路由构建 + 服务器启动
    session_actor.rs     — SyncWriter/SyncReader 桥接 + SessionActor（预留）
    frp.rs               — frp 客户端管理（配置生成、进程启停、TLS 支持）
```

## 3. 数据库扩展

### 3.1 新增表

```sql
-- 已配对设备
CREATE TABLE IF NOT EXISTS paired_devices (
    device_id TEXT PRIMARY KEY,
    device_name TEXT NOT NULL,
    token TEXT NOT NULL UNIQUE,
    paired_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_seen DATETIME DEFAULT CURRENT_TIMESTAMP

    key TEXT PRIMARY KEY,
```

## 5. 协议设计

### 5.1 WebSocket 消息格式

**服务端 → 客户端**：

```json
{"type": "pty_output", "seq": 1024, "data": "输出内容", "timestamp": 1718976000000}
{"type": "replay_complete", "replayed_count": 5}
{"type": "error", "message": "Session not found"}
```

**客户端 → 服务端**：

```json
{"type": "input", "data": "用户输入\r\n"}
{"type": "resize", "cols": 80, "rows": 24}
{"type": "replay", "last_seq": 1020}
```

### 5.2 认证流程

1. 桌面端生成 6 位 PIN（内存存储，5 分钟过期）
2. 手机 App 输入 PIN → POST `/api/pair/verify`
3. 服务端验证 PIN，返回 UUID v4 token
4. 后续请求通过 `Authorization: Bearer <token>` 头认证
5. Token 验证走内存 DashMap，不查 SQLite

## 6. 网络连接方式

支持四种连接方式，按推荐优先级排列：

| 方式 | 延迟 | 安全性 | 适用场景 |
|---|---|---|---|
| 局域网直连 | 最低 | 内网安全 | 同一 WiFi |
| 域名 + HTTPS + frp TLS | 低 | 加密 | 公网（推荐） |
| 域名 + HTTP + frp | 低 | 明文 | 公网（快速测试） |
| IP 直连 + HTTP | 低 | 明文 | 临时使用 |

### 6.1 局域网直连

```
手机 App → WS → 电脑 IP:9527
```

桌面端设置：启用远程访问，记下 IP:端口。

### 6.2 公网穿透（推荐）

```
手机 App → WSS(443) → nginx(SSL) → localhost:9527 ← frp隧道 ← 电脑
```

桌面端设置：启用远程访问 + frp 配置（服务器地址、端口、Token、TLS 开启）。
