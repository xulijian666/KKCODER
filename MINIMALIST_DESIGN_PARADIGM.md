# KKCoder 极简主义设计范式与前端改造自我检查清单 (Minimalist Design Paradigm & Audit Checklist)

> [!IMPORTANT]
> **设计纲领：极简、克制、极致易读。**
> KKCoder 是面向硬核开发者与高阶 AI 心流打造的客户端。一切界面元素必须摒弃廉价的玩具感，禁止使用原生 Emoji 作为主要 UI 元素，推崇高精度矢量 SVG 与无框扁平化（Flat Minimalist）交互。每次进行前端改造时，必须严格执行本规范！

---

## 💎 一、 极简主义核心设计原则

### 1. 禁用 Emoji 作为主要 UI 图标 (Emoji Ban in Primary UI)
* **规范**：主状态栏、侧边栏、对话标签、控制按钮、物理路径行、右键上下文菜单等全局交互组件中，**严禁出现原生彩色 Emoji 图标**（如 📂, 📋, 🗑️, 🖥️, ⚙️）。
* **替代方案**：统一使用线条精细、比例和谐的高保真单色 **SVG 矢量轮廓图标**。
  * 保持 SVG 样式属性简洁，strokeWidth 设为 `2` 或 `2.5`，尺寸限制在 `12px` ~ `14px`。
  * 图标主色应通过 `stroke="currentColor"` 继承主题色，或在 Hover 时优雅转换，杜绝静态的杂色斑驳感。
  * 针对特定的文件夹可以填充扁平化、低饱和度的纯色（如 `#EAB308` 奶油黄），但必须高度克制。

### 2. 极致克制的多主题色彩兼容 (Day/Night Theme Cohesion)
* **规范**：组件样式中**严禁硬编码任何绝对色彩值**（如纯白 `#ffffff`、纯黑 `#000000` 或强色调 `#3b82f6`），除非该色彩用于独立物理 Badge 或固定状态提示。
* **替代方案**：所有颜色必须无缝绑定 CSS 变量。任何新开发的组件必须完美适配 **6 套动态主题**（3 套白天模式，3 套黑夜模式）：
  * **白天模式**（经典高雅、冰蓝晴空、香草蜜橘）
  * **黑夜模式**（深蓝星空、暗紫午夜、黄铜碳黑）
* **绑定变量标准**：
  * **背景板/底板**：`var(--bg-main)` (主编辑区), `var(--bg-sidebar)` (侧边栏及弹窗底色)
  * **文字与前景色**：`var(--text-primary)` (主文字), `var(--text-secondary)` (辅助/说明文字)
  * **边框与虚线**：`var(--border-color)` (细分线)
  * **激活与悬浮**：`var(--bg-hover-item)` (悬浮柔和底), `var(--bg-active-item)` (选中高亮底), `var(--color-primary)` (主题焦点色)

### 3. 超扁平化与极度克制的按钮高度（Sleek & Slim Buttons）
* **规范**：状态栏、标签栏中的按钮不应拥有厚重的实体背景、坚硬的外部实体框线与复杂的阴影。
* **替代方案**：采用**透明背景与扁平圆角设计**。按钮常态下背景完全透明、无边框（`border: none !important; background-color: transparent !important;`）。
* **高度克制**：**按钮的上下高度必须极其克制，尽量与内部字体尺寸吻合，绝不能过高显得冗余 low**。
  * 设置切换组按钮、侧边栏新终端按钮、弹窗功能性按钮的垂直 Padding 严格控制在 `3px` ~ `5px` 内，Border-radius 统一限制在 `4px`，保持高水平贴合感与现代高级极简质感。

### 4. 物理挖空镂空 Badge 设计 (Cutout Overlay Badges)
* **规范**：当在头像、图标、按钮右上角叠加数字或状态角标（如 `.queue-badge` 队列角标）时，不能直接生硬地压在底层元素上。
* **替代方案**：为 Badge 元素设置与底层父级（如状态栏背景 `--bg-sidebar`）色彩一致的物理中空镂切边框（如 `border: 1.5px solid var(--bg-sidebar)`），在 Badge 与底层之间流出微小的空气 gap 镂空，使整个 UI 展现出一流的高档 3D 浮雕与层次视感。

### 5. 阻断任何布局重排抖动 (Zero-Jitter Selection & Stable Layouts)
* **规范**：禁止通过改变字体粗细（font-weight 变粗）、移动元素位置或根据动态数据改变容器高矮来响应用户点击/搜索操作，否则会导致页面发生高矮收缩、画面抖动（Layout Shift）。
* **替代方案**：
  * 交互列表及最近项目容器必须设定固定的高度与 `overflow-y`，杜绝内容动态渲染导致的容器高度跳动。
  * 点击选中项时，完全保持排列位置不动，仅改变背景高亮和字色，字体字重保持一致，实现物理零重排。

### 6. 标准的虚拟终端 IME 聚焦层物理隐藏规范 (Virtual Terminal IME Sandboxing)
* **规范**：为确保中文输入法（IME）选字候选框随命令行光标在打字时流畅、平滑地往右移，**严禁重写隐藏 textarea 的 width、height 和 left/top 定位属性**。
* **替代方案**：完全保留 `xterm.js` 标准内置的 cursor tracking 定位系统。我们仅在视觉层级（z-index）上将 `.xterm-helper-textarea` 置于 `-10 !important` 最底层，并结合 `background: transparent; border: none; outline: none;` 进行多维隐形，保证其在 Canvas 下方静默获取焦点及捕获系统输入定位。

## 🔍 二、 前端改造必备之「极简自我审查」环节 (Mandatory Review Checklist)

> [!IMPORTANT]
> 任何前端改造在提交或声明完成前，**必须作为世界级极简程序设计师，主动触发以下“自我审查环节”**，并逐项自我质询校验：

### ✍️ 自我质询标准卡片：
1. **[  ] 图标审查**：我是否在页面、按钮、控制条的任何角落随手打出了原生 Emoji 图标？如果有，我是否已经换成了单色且能自适应文字颜色的 `<svg>` 图标？
2. **[  ] 颜色审查**：我的 CSS 样式中是否存在类似 `color: #333` 或 `background: white` 等不兼容夜间模式的绝对代码值？如果有，我是否已经换成了符合设计体系 of `var(--text-primary)` 或 `var(--bg-main)`？
3. **[  ] 按钮高度克制审查**：我所设计的按钮（包括设置选卡、弹窗确认、侧栏按钮等）其**上下高度是否足够克制、贴合字体**？垂直 Padding 是否已控制在 `3px`-`5px` 极简范围内？Border-radius 是否均精细限制在 `4px`？
4. **[  ] 镂空审查**：我的右上角 badge 角标是否使用了 `border: 1.5px solid var(--bg-sidebar)`？如果没有，加上它之后是否能呈现惊艳的视网膜级物理隔离缺口？
5. **[  ] IME 候选框跟随性**：输入拼音时，输入框是否能顺畅、稳定地跟着光标向右边推移，且左边绝无剧烈抖动与偏移？
6. **[  ] 多主题盲测**：当在设置中切换到 “温暖香草奶油 (Vanilla Light)” 和 “暗紫午夜 (Midnight Purple)” 时，新增的元素在极白与极黑的背景下，前景色和 Hover 态的对比度是否均处于完美高易读区间？
